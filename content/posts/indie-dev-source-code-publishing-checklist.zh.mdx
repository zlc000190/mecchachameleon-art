# 独立开发者怎么发源码不踩 IP 雷 —— 粉丝自制项目的清单

> **提醒：** Meccha Chameleon Art Lab 是一个**非官方**、粉丝自制的伴生站点。它**不是**官方 Meccha Chameleon 游戏，也**不**与该游戏的开发商或发行商有任何关联。官方游戏在 [Steam](https://store.steampowered.com/app/4704690/MECCHA_CHAMELEON/) 上销售。

做 [mecchachameleon.art](https://mecchachameleon.art) 的时候，我们想发 Godot 模板、配色脚本、还有别的小代码片段——这些代码**致敬** Meccha Chameleon 这个类型，但绝不重新分发官方游戏的素材、美术或源码。这份清单就是我们希望第一天有人递给我们的那一张。

不是法律建议。它是一组务实的习惯，这些习惯让我们过去一年和 Meccha Chameleon 社区以及更广的 Godot 社区都保持良好关系。

## 发布前 7 问清单

在你 push 任何提到商业游戏的东西之前，先走一遍：

### 1. 这个产物**衍生自**该游戏的代码或素材吗？

如果是，**停**。你不能以任何协议发布商业游戏代码或美术的衍生品。在很多司法管辖区，反向工程用于互操作是合法的，但发布结果代码不合法。

如果否，你发的是**受类型启发、原创**的代码，不是衍生品。继续。

### 2. 这个产物里有没有游戏的美术、音效、字体或商标？

如果有，剥掉换成占位。即使是 README 里一张官方游戏的截图也是个风险——它暗示了背书。

如果否，继续。

### 3. 这个产物有没有点名该游戏？

如果有，用商标免责声明模式：

> "Meccha Chameleon 是 [持有人] 的商标。本项目为非官方、粉丝自制作品，与 [持有人] **无任何关联**。"

如果否，你可以省略免责声明，但考虑一下"不提游戏"会不会削弱教育价值。

### 4. 这个产物能不能在没有该游戏任何文件的情况下独立运行？

如果能，你就稳。如果不能，你在重新分发部分副本，那是更难辩护的位置。

### 5. 这个产物有没有协议？

如果**没有**，**加一个明确的协议**。GitHub 上无协议的代码在大多数司法管辖区默认是"版权所有"——这和"粉丝自制学习项目"的精神矛盾。小模板我们用 MIT。

如果有，继续。

### 6. 这个产物里有没有用任何**非宽松协议**的第三方素材？

检查 Godot Asset Library 条款、Kenney.nl 条款、OpenGameArt 协议。如果你拿了 CC-BY-NC 的素材，就不能在还有商业潜力的项目里用。剥掉或替换。

### 7. 这个产物有没有带埋点、分析或自动更新？

如果有，粉丝自制项目里直接去掉。埋点尤其会制造一个隐私风险面，而粉丝对"就一个学习仓库"不会有这种预期。

## 我们**不发**的东西

- **反编译或反向工程的 Meccha Chameleon 代码。** 我们没有，也不会发。
- **官方游戏的素材撕取**（精灵、音效、音乐、字体）。我们只用自己拍的截图，且仅用于合理使用范围内的编辑性评论。
- **商标抢注。** 我们不注册包含 "Meccha Chameleon" + 通用后缀（比如 `.com`、`.net`、`.io`）的域名。我们唯一的域名是 `.art`，且明确框定为粉丝自制实验室。
- **付费课程、付费素材、或者用商标的周边**。只做免费学习材料。

## 如果权利人要求你下架，怎么办

- 快速配合。不要先在公开场合吵。
- 存档请求（带时间戳）给自己留底。
- 如果你认为请求是错的（比如把你和别人搞混了），简短地回应一次，给事实。不要升级到社交媒体。
- 下架后，发一个简短的公开说明："该产物应权利人要求已移除。" 这能保护未来的贡献者不重复同样的错误。

## 为什么要分享这份清单

大多数"怎么发源码"指南假设你在做自己的原创 IP。粉丝自制的情况更乱：你想教东西、想引用一款真实游戏来给上下文，但又不想踩商标。这些规则是我们试错学到的，把它们公开，能给下一个人省几个月。

## 怎么贡献

如果你维护类似的粉丝自制项目，并且对清单有补充（特别是非美国贡献者的司法管辖说明），去 [`CONTRIBUTING.md`](https://github.com/zlc000190/mecchachameleon-art/blob/main/CONTRIBUTING.md) 开 PR。**匿名提交欢迎**。

如果你维护一款商业游戏，希望我们从仓库里移除对你作品的引用，发邮件到 `support@mecchachameleon.art`。几天内回复。
